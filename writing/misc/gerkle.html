gerkle

I've been reading a lot about crypto lately and in the bitcoin whitepaper the way the blockchain saves space is by using a merkle tree. They don't look that hard but I was thinking I was struggling to understand how it works so I decided to implement my own (from scratch... pretty much). I'm going to call it gerkle because 1- it's catchy 2- it's not taken on the go package registry (aka github). One of the rules I'm setting for myself is that I won't look at any other implementations in Go (all my sources will be listed below). The reason for this is that I want to improve my Go and make sure the concept really sinks in.
